## Summary
The mem_bank branch evolved the session-summary capture system from a single monolithic hook script into a modular, multi-file subsystem. Early work fixed a critical fork-bomb in the SessionEnd hook, where each spawned `claude -p` worker triggered its own SessionEnd, cascading infinitely; the fix introduced a `MEM_BANK_HOOK_RECURSION_GUARD` environment variable checked at hook entry so child invocations short-circuit before forking. Prompt-slimming was added to keep token costs low: skill bodies collapse to elision markers, long prompts are head/tail-truncated, and harness noise tags are stripped, while user-typed input is always preserved verbatim because voice-dictated prompts are considered irreplaceable human signal. A subscriptions.json file was introduced so banks declare which transcript patterns they listen to, supporting both a single `pattern` key and a `patterns` list for multi-regex matching, enabling future mem-banks across different project areas to each self-describe their triggers. The directory structure was reorganized from scattered locations into a unified `.claude/mem-bank/` directory and the naming was clarified: `append-session-summary.py` became `small-bank.py` (capture), `graduate.py` became `big-bank.py` (archive), and shared logic — Claude API calls, logging utilities, bank discovery — was extracted into `utils.py` and `registry.py` to eliminate duplication across the three entry-point scripts. Big-bank graduation was also restructured so small-bank contents are stored in a `small-bank-archive/` subdirectory rather than appended inline, keeping the main big-bank files lean and avoiding context bloat. A pre-merge-commit hook was wired to check for unpopulated banks before merges, and a git alias enforces no-fast-forward merges to ensure merge commits are always created.
